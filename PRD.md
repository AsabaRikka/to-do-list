# To-Do List 产品需求文档 (PRD)

## 1. 产品愿景
打造一款极简、高效、跨平台的待办事项管理工具，深度致敬 Microsoft To-Do 的核心交互与视觉设计，解决用户在不同场景（Web、手机、电脑）下的任务管理需求。

## 2. 目标平台
- **Phase 1 (MVP)**: Web 端 (响应式设计)
- **Phase 2**: 微信小程序 (满足移动端即时查看与提醒)
- **Phase 3**: macOS原生/跨平台客户端 (提供更深度的系统集成)

## 3. 核心功能需求 (MVP 阶段)

### 3.1 任务基础管理
- **创建任务**: 支持快速输入任务标题。
- **任务状态**: 完成/未完成 切换。
- **星标重要性**: 标记重要任务，并在“重要”列表中汇总。
- **删除任务**: 物理/逻辑删除。

### 3.2 预设智能列表
- **我的一天 (My Day)**: 每天清空的专注列表，用户手动添加当天的任务。
- **重要 (Important)**: 汇总所有标记为星标的任务。
- **计划内 (Planned)**: 按截止日期排序的任务汇总。
- **任务 (Tasks)**: 默认收纳箱。

### 3.3 任务详情 (侧边栏)
- **步骤 (Sub-tasks)**: 一个大任务下的细分操作步骤。
- **添加到“我的一天”**: 快速关联。
- **提醒我**: 设置具体提醒时间。
- **添加截止日期**: 设置任务到期日。
- **重复**: 支持每天、每周、每月、自定义重复。
- **备注**: 支持多行文本备注。

### 3.4 列表管理
- **新建列表**: 用户自定义分类（如：工作、生活、学习）。
- **列表分组**: 将多个列表归类到一个组内。

## 4. 技术架构建议 (为多端做准备)

为了满足后续微信小程序、iOS 和 macOS 的开发需求，建议采用以下架构：

### 4.1 后端 (API First)
- **技术选型**: Python (FastAPI/Django) 或 Node.js (NestJS)。
- **核心能力**: 
  - 提供标准的 RESTful API 或 GraphQL 接口。
  - 统一的用户鉴权 (JWT/OAuth2)。
  - 数据库支持云端同步 (PostgreSQL/MongoDB)。

### 4.2 前端 (Web MVP)
- **技术选型**: React + Tailwind CSS + Vite。
- **优势**: 
  - 开发效率高，组件化程度高。
  - Tailwind 方便快速复刻 Microsoft To-Do 的简洁 UI。
  - 逻辑层可部分复用于后续移动端。

### 4.3 跨平台演进路径
- **微信小程序**: 使用 **uni-app** 或 **Taro**。这样可以利用 Vue/React 的开发经验，一套代码适配小程序和 iOS/macOS (通过 Webview 或原生桥接)。
- **iOS/macOS**: 
  - 方案 A: **React Native / Expo** (共享大部分 Web 业务逻辑)。
  - 方案 B: **Electron** (仅限 macOS，快速将 Web 版转桌面版)。
  - 方案 C: **Swift/SwiftUI** (针对苹果生态做极致体验，仅共享后端接口)。

## 5. UI/UX 设计要点
- **左侧导航栏**: 清晰的智能列表和自定义列表分类。
- **中间任务列表**: 宽敞的点击区域，支持拖拽排序（后期）。
- **右侧详情面板**: 点击任务弹出，不干扰主视图。
- **配色**: 经典微软蓝 (Microsoft Blue) 为主色调，支持更换背景主题。

## 6. 路线图 (Roadmap)
1. **Week 1-2**: 完成 Web 端基础 CRUD 和核心智能列表。
2. **Week 3**: 实现任务详情（提醒、重复、步骤）。
3. **Week 4**: 用户登录与云端同步。
4. **Week 5**: 启动微信小程序版本。
